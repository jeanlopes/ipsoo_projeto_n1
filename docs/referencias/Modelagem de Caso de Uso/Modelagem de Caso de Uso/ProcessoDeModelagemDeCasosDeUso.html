    • + - Recomendações

      • 1 - Apenas casos de uso preliminares

        2 - Não há dependedência temporal entre casos de uso

        3 - Atores do Cliente envolcem-se em vários casos de uso

        4 - Casos de uso NÃO são funções

        5 - Não há setas nas associações

        6 - Diagrama deve ser pequeno

        7 - Casos de uso devem ser funcionalmente coesos

        8 - Cada caso de Uso deve ser temporalmente coeso

        9 - Casos de uso devem descrever algo de valor para o negócio

        10 - Todo ator deve estar envolvido envolvido com pelo menos um caso de uso e cada caso de uso deve estar envolvido com pelo menos um ator

        11 - Ações repetitivas não precisam ser expressas em um único caso de uso

        12 - Limite do sistema é Opcional

    • + - Exemplo
      • + - Informal

        • Nome: Enroll in Seminar

          Identificador: UC 17

          Fluxo Principal:

          • Aluno entra seu nome e número de matricula

          • Sistema verifica a eligibilidade do aluno para matricula no seminário. Se não for elegível, aluno é informado e CDU termina.

          • Sistema mostra a lista de seminários disponíveis.

          • Estudante escolhe um seminário e decide se irá se matricular.

          • Sistema valida se o aluno é elegível para matricular-se no seminário escolhido. Se não for, é pedido que esolha outro.

          • Sistema valida se o seminário se encaixa na agenda do aluno.

          • Sistema calcula e mostra as taxas.

          • Aluno verifica o custo e indixa se deseja se matricular ou não

          • Sistema matricula aluno no seminário e registra as taxas.

          • Sistema imprime o recibo de matrícula.

      • + - Formal

        • Nome: Matricular no Seminário

          Identificador UC 17:

          Descrição:

          Matricular um aluno existente num seminário que esteja elegível

          Pré-condições:

          O aluno está registrado na universidade.

          Pós-condições:
          O aluno estará matriculado no curso que deseja se estiver elegível e se tiver vagas.

          Fluxo Principal:

          1. O caso de uso inicia quando o aluno indica que deseja se matricular no seminário.

          2. O aluno entra seu nome e matrícula no sistema via UI23 Tela de Login.

          3. O Sistema verifica se o aluno é ellegível para matrícula nos seminários na universidade de acordo com a regra de negócio RN129 Determinar elegibilidade para matrícula. [Fluxo alternativo A]

          4. O sistema mostra UI32 Tela de Seleção de Seminário, que indica a lista de seminários dispníveis.

          5. O aluno indica o seminárqui que deseja se matricular.[Fluxo alternativo B: O Aluno decide não se matricular]

          6. O sistema valida se o aluno é elegível para matrícula em um semin;ario de acordo com a regra de negócio RN130 Determinar Eligibilidade do ALuno para se Matricular no Seminário. [Fluxo alternativo C]

          7. O Sistema valida se o seminário se enquadra na agenda existente do aluno de acordo com a regra de negócio BR143 Validar Agenda do Aluno.

          8. O Sistema calcula as taxas para o seminário baseado nas taxas publicadas no catálogo de cursos, aplicar as taxas do aluno e aplicar impostos, Aplicar RN180 Calcular Taxas do ALuno e BR45 Calcular Impostos para o Seminário.

          9. O Sistema mostra as taxas via UI33 Mostrar Taxas do Seminário.

          10. O Sistema pede ao aluno sua confirmação  de matrícula no seminário.

          11. O Aluno indica que deseja se matricular no seminário.

          12. O sistema matriculao aluno no seminário.

          13. O sistema informa ao aluno que a matrícula foi um sucesso via UI88 Sumário de Matrícula em Seminário

          14. O sistema cobra o estudante pelo seminário, de acordo com a regra de negocio RN100 Cobrar aluno pelo seminário.

          15. O sistema pede que aluno se o aluno quer imprimir o comprovante de matrícula

          16. O aluno indica que deseja imprimir o comprovante

          17. O sistema imprime o comprovante de matrícula UI89 Relatório de Sumário de Matrícula.

          18. O caso de uso termina quando o aluno pega o comprovante impresso.

          Fluxo Alternativo A: O aluno não é elegível para matricula nos seminários

          A.3. O secretário determina que o aluno não é elegível para matricular-se no seminário.

          A.4. O secretário informa que aluno não é elegíevl para matrícula.

          A.5. O caso de uso termina.

          Fluxo Alternativo B: O aluno decide se não se matricula em um seminário disponível

          B.5. O aluno vê a lista de seminários e não vê o que deseja se matricular.

          B.6. O caso de uso termina.

          Fluxo Alternativo C: O aluno não tem os pré-requisitos

          C.6. O secretário determina se o aluno não é elegível para matrícula no seminário escolhido.

          C.7. O secretário informa ao aluno que ele não possui os pré-requisitos

          C.8. O secretário informa ao aluno sobre os pré-requisitos que precisa.
